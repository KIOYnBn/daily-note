## 蛋白质数据库
### 一级序列数据库
* uniprot
	* 组成： 
		* Swissprot： 可信度高， 冗余度小
		* TrEMBL： 计算机完成。可信度低， 冗余大
		* PIR： 
	* 层次：

#### InterPro
| 对比维度     | **InterPro**                   |
| -------- | ------------------------------ |
| **核心类型** | **蛋白质序列功能综合库**                 |
| **主要目标** | **功能预测**：给任意蛋白序列打上“功能标签”       |
| **数据来源** | 集成13个成员数据库的**预测模型（特征签名）**      |
| **构建方式** | **人工审编+自动化整合**                 |
| **输出结果** | 家族、结构域、功能位点、GO术语等              |
| **优势**   | **覆盖面广、功能全面、实用性强**，是大规模序列注释的首选 |

## 专用数据库
### KEGG
|对比维度|**KEGG**|**InterPro**|**CATH / SCOP2**|
|---|---|---|---|
|**核心类型**|**生物系统功能综合库**|**蛋白质序列功能综合库**|**蛋白质结构域分类库**|
|**主要目标**|**系统级功能解析**：揭示基因、化合物如何在通路和网络中协同工作|**功能预测**：给任意蛋白序列打上家族、结构域等“功能标签”|**结构分类**：为已知结构的蛋白质域进行层级分类，揭示进化关系|
|**核心单元**|**KEGG Orthology (KO)**，即**K编号**|**InterPro Entry**（整合的蛋白家族/结构域）|**结构域**（如CATH的H层级、SCOP2的超家族）|
|**特色输出**|PATHWAY通路图、BRITE功能层级、疾病-药物网络|功能位点、GO功能注释|结构分类号、结构进化关系图谱|
|**一句话定位**|告诉你基因在**细胞中“做什么”**（处于哪条通路）|告诉你基因**“是什么”**（属于哪个家族）|告诉你基因的**“长相”**（三维结构如何折叠）|
