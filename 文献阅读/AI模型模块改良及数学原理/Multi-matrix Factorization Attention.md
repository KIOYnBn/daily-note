# 目的
* 减少注意力机制对内存的占用过大问题
# 背景
1. KV 缓存
	1. LLM（如 GPT 系列）采用自回归解码，每次生成新 token 时，需要复用之前所有 token 的**键（Key）和值（Value）张量**（即 KV 缓存），避免重复计算。但 <font color=green>KV 缓存的内存占用会随「批量大小」和「序列长度」线性增长</red>，是解码阶段的主要内存瓶颈 
2. 现有方法的缺陷
	1. 之前的方法（MQA、GQA、MLA）通过「共享投影参数」或「低秩压缩」减少缓存，但存在核心缺陷：
		- **性能妥协**：为了省缓存，限制了注意力头的表达能力（如 MQA 共享所有头的 KV 投影），导致性能不如 MHA；
		- **兼容性差**：如 MLA 需要额外修改才能适配主流位置编码（如 RoPE），难以集成到现有 LLM 生态；
		- **容量不足**：现有方法未充分利用「注意力头的数量和维度」对模型容量的影响，导致容量上限低于理论值。
3. 注意力容量的关键影响因素
	1. **容量上限基准**：全参数双线性注意力（FPBA）是 GMHA 框架下的容量上限 —— 它允许每个通道独立交互、不压缩维度，能实现无限制的特征交互；
		1. MHA attention: $O_i = \sum_{c=1}^n \left( \sum_{j=1}^i \phi \left( \frac{x_i Q_c (x_j K_c)^T}{\sqrt{d}} \right) x_j V_c \right) O_c^T$
		2. FPBA attention: $O_i = \sum_{c=1}^{H} \left( \sum_{j=1}^{i} \phi \left( \frac{x_i W_c x_j}{\sqrt{H}} \right) x_j U_c \right)$
			1. $W_c$ 拆分成 $W_q$、$W_k$
			2. $U_c$ 拆分成 $V_c$、$O_c$
	2. **容量关键因子**：注意力模型的容量由「总有效秩（TER）」决定，TER = 头的数量 × 每个头的分解秩（FRH）。之前的方法（MQA、GQA、MLA）要么减少头数，要么降低分解秩，导致 TER 偏低，性能不如 MHA。

* <font color=green>因此，论文的核心思路是：在严格控制 KV 缓存的前提下，通过高效扩展头的数量和维度，提升 TER，让模型容量接近 FPBA 的上限。</font>

## MFA
- 「扩展头的数量和维度」：增加头数（m > n，n 为 MHA 的头数），并提升每个头的分解秩（C > d，d 为 MHA 的头维度），直接提升 TER；
- 「低秩矩阵分解」：在 QK 电路中对权重矩阵 Wc（负责特征交互）和转换矩阵 Uc（负责特征输出）进行低秩分解（Wc=Sq・Qc・Sk^T，Uc=Sv・Oc^T），避免参数爆炸；
- 「单键值头技术」：所有注意力头共享同一组 KV 投影（Sq、Sk、Sv），KV 缓存仅与低秩维度 C 相关（每 token 缓存 = 2C），不随头数增加而增长。
## MFA-KR
复用键缓存（Key Cache）作为值（Value），通过「重参数化的值投影」实现：

- 将值投影 Sv 重参数化为：Sv = Sk ⊙ α + N（α 是零初始化的门控向量，N 是可学习矩阵，⊙是元素积）；
- 训练初期 α=0，确保 Sv=Sk，保证训练稳定性；训练后期 α 自适应调整，平衡缓存效率和性能。
