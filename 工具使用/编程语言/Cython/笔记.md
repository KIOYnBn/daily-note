## 一、文件后缀说明

|            |              |                                               |
| ---------- | ------------ | --------------------------------------------- |
| 后缀         | 用途           | 说明                                            |
| **`.pyx`** | Cython 实现文件  | **最常用**，写核心代码（Python 语法 + C 类型注解），会被编译成 C 代码  |
| `.pxd`     | Cython 头文件   | 类似 C 的 `.h` 文件，声明 C 类型、函数、类接口，供其他 `.pyx` 文件导入 |
| `.pyi`     | Python 存根文件  | 给 IDE 提供类型提示，不参与编译，可选                         |
| `.py`      | 普通 Python 文件 | Cython 也可以直接编译纯 `.py` 文件，但无法使用 C 类型注解，加速效果有限  |

## 二、标准编译流程
### 1. 前置依赖
```bash
# 安装 Cython 和编译工具链
pip install cython setuptools wheel

sudo apt install gcc
```
### 2. 编写 `setup.py` 编译脚本
```python
from setuptools import setup
from Cython.Build import cythonize

setup(
    name="你的模块名",
    version="1.0",
    # 核心：cythonize 接收要编译的文件列表
    ext_modules=cythonize(
        "你的文件名.pyx",  # 单个文件
        # ["file1.pyx", "file2.pyx"],  # 多个文件
        compiler_directives={
            "language_level": "3",  # 强制使用 Python 3 语法（必加）
            "boundscheck": False,   # 关闭数组越界检查（加速，生产环境建议开启）
            "wraparound": False     # 关闭负索引支持（加速）
        }
    )
)
```
### 3. 执行编译命令
```bash
# 1. 彻底清理旧编译文件（每次修改代码后必须执行！）
rm -rf build *.so *.o *.c

# 2. 编译生成扩展模块（--inplace 表示生成到当前目录）
python setup.py build_ext --inplace
```
### 4. 编译产物
- Linux/macOS：生成 `模块名.cpython-3xx-x86_64-linux-gnu.so`
- Windows：生成 `模块名.cp3xx-win_amd64.pyd`
### 5. 使用模块
```python
import 你的模块名
print(你的模块名.函数名(参数))
```
## 三、快速编译方式（适合测试）
### 1. 命令行直接编译（无需 [setup.py](setup.py)）
```bash
# 编译单个 .pyx 文件
cythonize -i 你的文件名.pyx
```
- `-i` 等价于 `--inplace`，生成到当前目录
### 2. Jupyter Notebook 魔法命令
```python
%load_ext cython

%%cython
def add(int a, int b):
    return a + b

print(add(3, 5))  # 直接运行
```
## 四、完整示例
### 1. `demo.pyx`
```cython
def sum_n(int n):
    cdef int total = 0
    cdef int i
    for i in range(n):
        total += i
    return total
```
### 2. `setup.py`
```python
from setuptools import setup
from Cython.Build import cythonize

setup(
    name="demo",
    ext_modules=cythonize("demo.pyx", compiler_directives={"language_level": "3"})
)
```
### 3. 编译并运行
```bash
rm -rf build *.so && python setup.py build_ext --inplace
python -c "import demo; print(demo.sum_n(1000000))"
```
## 五、常用编译选项

|   |   |
|---|---|
|选项|作用|
|`--inplace`|生成扩展模块到当前目录（必加）|
|`-O3`|开启最高级编译优化（在 [setup.py](setup.py) 的 extra_compile_args 中添加）|
|`-march=native`|针对当前 CPU 架构优化（进一步加速）|
|`--annotate` / `-a`|生成 HTML 性能报告（查看哪些代码是 Python 交互）|

### 带优化的 [setup.py](setup.py) 示例
```python
from setuptools import setup, Extension
from Cython.Build import cythonize

ext_modules = [
    Extension(
        "demo",
        ["demo.pyx"],
        extra_compile_args=["-O3", "-march=native"]
    )
]

setup(
    name="demo",
    ext_modules=cythonize(ext_modules, compiler_directives={"language_level": "3"})
)
```
## 六、关键注意事项
1. **缓存问题**：每次修改 `.pyx` 文件后，必须先 `rm -rf build *.so` 再重新编译，否则会运行旧代码
2. **Python 版本匹配**：用哪个 Python 版本编译，就只能用哪个版本导入
3. **C 编译器**：Linux 用 GCC，Windows 用 MSVC（需安装 Visual Studio 生成工具）
4. **性能分析**：执行 `cython -a demo.pyx` 生成 `demo.html`，黄色越深表示 Python 交互越多，是优化重点
5. **不要返回 C 指针**：Python 无法理解 C 指针，需转换为 Python 列表 / NumPy 数组
## 七、速查命令

```bash
# 安装依赖
pip install cython

# 编译（标准方式）
rm -rf build *.so && python setup.py build_ext --inplace

# 编译（快速方式）

# 生成性能报告
```
